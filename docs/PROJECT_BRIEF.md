# PROJECT_BRIEF.md

## Project Goal

The goal of Afif Portfolio is to support three main directions:

- Job opportunities in Front-End / React development
- Freelance client opportunities
- Personal brand building

The portfolio should not feel like a simple resume website. It should present Afif as a creative and technically strong Front-End Developer who can build modern, polished, and business-aware web experiences.

The portfolio should feel custom-built and memorable, with a cinematic command-interface style rather than a generic template or strict Figma-to-code implementation.

---

## Positioning

Afif should be positioned as a:

**Creative Front-End Developer with a business mindset**

The portfolio should communicate both technical ability and product/business awareness.

It should show that Afif is not only able to implement UI, but also able to think about user experience, presentation, performance, and business value.

---

## Target Audience

The portfolio should speak to multiple audiences:

- Developers
- Technical recruiters
- HR teams
- Freelance clients
- Potential collaborators

The primary audience is developers.

This means the website should feel technically credible, well-structured, and clearly implemented, not only visually attractive.

---

## Desired First Impression

Within the first 10 seconds, the visitor should feel that the portfolio is:

- Technically strong
- Creative
- Professional
- Different from generic template portfolios
- Visually memorable without feeling over-designed

The website should feel custom-built, not copied from a common portfolio template.

The experience should feel like entering a personal creative interface, not browsing a standard resume page.

---

## Current Design Direction

The current portfolio direction is:

**Custom cinematic command-interface portfolio**

The design is no longer a strict Figma-matching implementation. Figma files and visual references may still be used as inspiration, but the approved current style and user feedback are the source of truth.

The visual direction should feel:

- Dark navy / near-black
- Premium
- Cinematic
- Futuristic
- Technical
- Clean
- Custom-built
- Interface-driven

Key design elements may include:

- Global dark navy grid background
- Soft cyan/electric-blue glow spots
- Thin frame details and corner marks
- Command Gate intro screen
- Fullscreen split navigation menu
- Custom cursor interactions
- Controlled premium motion

Do not copy visual references directly. Use them only as inspiration when they support the current identity.

---

## Key Strengths to Highlight

The portfolio should strongly highlight:

- Animation
- UI/UX sense
- Performance
- Creative front-end implementation
- Business-aware thinking

These strengths should appear through both the content and the actual implementation quality.

---

## Experience Layer

The portfolio may include a short cinematic entry experience before the main content.

Current approved intro direction:

- Command Gate + Loading Experience
- Shows `AFIF.GH`
- Shows `CREATIVE FRONT-END DEVELOPER`
- Shows `Code is not just built. It is crafted.`
- Displays a short `INITIALIZING INTERFACE...` loading sequence
- Shows `SYSTEM READY`
- Reveals the CTA: `UNLOCK THE EXPERIENCE →`
- On click, uses a cinematic vertical screen-opening transition

The intro should be short, premium, and skippable through a clear CTA. It should not feel slow, childish, or distracting.

The intro is an experience layer, not a replacement for the main portfolio content.

---

## Main Sections

The portfolio should include the following main content sections:

1. Navbar / Menu
2. Hero
3. About
4. Experience
5. Projects
6. Skills
7. Contact
8. Footer

Do not add extra main content sections unless explicitly requested.

The website should stay focused, polished, and purposeful.

---

## Core Message

The portfolio should communicate a strong and confident message:

**I build modern front-end experiences that combine clean UI, purposeful motion, strong performance, and business-aware thinking.**

Alternative shorter version:

**Modern front-end experiences with clean UI, purposeful motion, and business-aware execution.**

The final wording can be refined during the Hero section implementation, but the message should stay focused on:

- Front-end development
- UI/UX quality
- Animation
- Performance
- Business mindset

---

## Business Background

Afif’s Business Administration background should be mentioned explicitly.

It should not dominate the portfolio, but it should support the positioning.

The business background should help communicate that Afif understands not only how to build interfaces, but also how digital products support users, clients, and business goals.

This can appear mainly in the About section and optionally in a short Hero supporting line.

---

## Content Tone

The content should feel confident and strong.

Avoid weak, generic, or overly humble language.

Also avoid exaggerated claims or unrealistic self-promotion.

The tone should be:

- Confident
- Clear
- Professional
- Direct
- Modern
- Slightly cinematic when appropriate

The content should support the interface style without becoming vague, theatrical, or over-marketed.

---

## What to Avoid

The portfolio should avoid:

- Generic resume-style portfolio structure
- Too much text
- Fake or exaggerated claims
- Template-like sections
- Overly corporate tone
- Too playful tone
- Too much focus on certificates
- Too much focus on personal story
- Copying reference designs directly
- Overusing cinematic effects at the expense of usability
- Making the interface feel like a game instead of a professional portfolio

The website should feel focused, custom-built, confident, and technically credible.
